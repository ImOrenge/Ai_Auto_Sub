0) 랜딩의 목표/원칙

1차 목표: “무료로 시작” 클릭(회원가입) / “로그인” 완료

2차 목표: 기능 신뢰(자동화 파이프라인) + 결과물 품질(자막 샘플) + 가격 납득

원칙

첫 화면에서 “이게 뭔지” 3초 안에 이해

스크롤 내릴수록 “의심 → 확신”으로 전환(증거/샘플/사회적 증명/FAQ)

CTA는 항상 1개가 주인공(Primary), 나머지는 보조(Secondary)

1) 페이지 전체 레이아웃(권장 IA)
A. Sticky Header (상단 고정)

구성

좌: 로고 AI Sub Auto

중: 메뉴 Features / How it works / Pricing / FAQ

우: 로그인(텍스트 버튼) + 무료로 시작(Primary 버튼)

동작

로그인 상태면 우측 CTA를 대시보드로로 변경

스크롤 시 헤더는 높이 축소 + 배경 블러(고급 SaaS 느낌)

2) Hero (첫 화면) — “한 문장 가치 + 즉시 행동”
Hero 레이아웃(2열)

좌측(카피)

Eyebrow: AI Subtitle Automation

H1: 업로드만 하면 자막이 자동으로 완성됩니다.

Sub: STT → 번역 → 편집 → SRT/MP4 내보내기까지, 작업 큐로 한 번에 처리.

CTA:

Primary: 무료로 시작하기

Secondary: 데모 보기(모달/스크롤 앵커)

보조 정보(신뢰):

한국어 포함 XX개 언어 / SRT·VTT·MP4 / 대량 처리(멀티잡)

우측(비주얼)

제품 스크린샷(대시보드/Jobs 그리드/자막 에디터)

“진행중 상태” 칩(UPLOADED, STT, TRANSLATE, AWAITING_EDIT, DONE)로 자동화 느낌 강화

✅ 너가 말한 “title”은 사실상 Hero의 H1로 흡수하는 게 자연스럽고 전문적으로 보임.

3) Title + Login UI (권장 방식: “임베드 로그인 카드”는 선택)

랜딩에 로그인 UI를 “박아두면” 전환에 좋을 때도 있지만, B2B SaaS 느낌은 보통 아래 둘 중 하나가 더 깔끔해.

옵션 1) 로그인은 모달/별도 페이지(가장 SaaS스러움)

Hero 우측은 제품 이미지에 집중

헤더의 로그인 클릭 → 모달 또는 /login

옵션 2) Hero 아래 “Start box”(가벼운 임베드 카드)

이메일로 시작 입력 + 버튼 하나

비밀번호/소셜로그인은 다음 단계로 넘김(마찰 최소화)

추천: 옵션 1 (랜딩이 “마케팅 페이지”처럼 더 고급스럽게 보임)

4) Social Proof (짧게, 하지만 강하게)

Hero 바로 아래에 얇은 줄 형태로:

Used by teams at … (로고 4~6개) 또는

지표형: 월 X시간 절약 평균 작업시간 Y% 단축 N개의 작업 동시 처리

5) Marketing 섹션(= Value/Proof 섹션을 3덩어리로 쪼개기)

너가 말한 “marketting”을 그냥 글로 두면 약해져서, 아래 3파트로 나누면 랜딩 완성도가 확 올라가.

5-1) Problem → Solution (Before/After)

Before: “자막 제작은 반복 + 사람 손이 많이 감”

After: “업로드/큐 관리로 자동 생성 + 필요한 부분만 편집”

시각: 좌우 비교 카드(작업시간/단계/오류)

5-2) How it Works (파이프라인)

4단계 타임라인:

Upload

STT

Translate

Edit & Export

각 단계마다 “산출물/상태/예외처리”를 한 줄로 명확히

5-3) Feature Grid (6~9개)

예시 카테고리(너 서비스랑 잘 맞음)

Multi-job Queue: 여러 영상 동시에 처리/대시보드 그리드

Subtitle Editor: 스타일/타임라인/미리보기

Export: SRT/VTT/MP4 Burn-in

Team & Roles: 협업/권한(추후)

Templates: 프리셋/이펙트(추후 계획 포함)

API/Webhook: 자동 연동(추후)

Security: 파일 보관/삭제 정책, 접근제어

Quality: 용어사전/커스텀 단어(추후)

6) PlanList (Pricing) — “표 + 비교 + 사용량 모델” 3단 구성

가격은 랜딩 신뢰의 핵심이라, “플랜 카드만 3개”로 끝내면 아쉬워.

6-1) Pricing Toggle

월간 / 연간(할인) 토글

“대부분 연간 선택” 배지로 유도

6-2) Plan Cards (3~4개)

Free / Starter / Pro / Team(또는 Business) / Enterprise(문의)

각 카드에:

대상(누구에게 좋은지)

핵심 가치 3개(말로)

포함 사용량(분/작업수/저장기간)

초과 과금(사용량 기반이면 명시)

6-3) Feature Comparison Table (아래에 1개)

카드 아래에 비교표를 넣으면 “결정”이 쉬워짐

팁: “자막은 결국 분(minutes) 과금”이 설득력이 좋아서, Usage 기반이면 랜딩에 미리 보여주는 게 전환에 유리해.

7) FAQ (반대질문을 미리 제거)

FAQ는 최소 6개만 깔끔하게:

정확도/지원 언어

긴 영상/대량 처리 가능?

편집 툴 범위(스타일/프리셋)

파일 보관/삭제 정책

환불/플랜 변경

저작권/보안
